Working with OpenDDS Models

The OpenDDS Model

The OpenDDS Model consists of "libraries" of possibly re-usable OpenDDS elements and optionally Packages that can contain these libraries.

Library in this context is different than the concept of C++ object file libraries used in OpenDDS. Even though you may be working with different modeling libraries the generated code will target a single object file library.

Creating a Model

To create an OpenDDS SDK model, from the Eclipse File -> New -> Other... (Ctrl+N) menu item select OpenDDS Diagram under the OpenDDS Models category as shown in the figure below.

The Create OpenDDS Diagram wizard when then prompt for the name of the diagram file and the name of the file containing domain model itself. It is recommended that when specifying the model file name you use the default which has the same base name as the diagram file.

Many diagram-based modeling tools keep both the model and diagram elements in a single file. In the case of the OpenDDS SDK these files are kept separate. This will readily allow for alternative representations of the model that coexist with diagrams to be considered.


After selecting the name of the diagram file, make sure you select the Next button instead of Finish. Under certain conditions the base portion of default name for the model file will not match the base portion of the diagram file if this step is skipped.

Populating the Model
